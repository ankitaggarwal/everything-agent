Every box is a real part of the system. The two lanes in Act are the whole point: the instant lane keeps the robot snappy, the agent lane does real work with tools. The router decides which — in well under a second.

The two paths · why it stays fast

⚡ Instant lane

The router answers directly with a small fast model — no tools, no planning. "What time is it?", "Tell me a joke", "Look left", "Stop". Feels like talking to a person.

🤖 Agent lane

The router escalates to the Claude Agent SDK, which plans and calls tools — mostly MCP servers. Slower, but it can actually order food, print, research, control your home. Sensitive steps pass the approval gate first.

Things to design for (easy to forget)

✅ Approval gate.

Ordering food, sending messages, buying — anything that spends money or can't be undone asks you first. Built in as its own step, not bolted on.

🧠 Memory.

A personal robot should remember your name, "the usual", ongoing projects, who's who. Short-term (this chat) + long-term (a store it can search).

🗣️ Barge-in.

You can interrupt it mid-sentence; it stops talking and listens. Essential for it to feel natural.

🔒 Privacy.

Wake-word detection runs locally so the mic isn't streaming to the cloud all day. Be explicit about what leaves the device — matters doubly for open source.

Where this lives in the code

everything_agent/
├── agent.py           # the always-on loop (hear → decide → act)
├── core/
│   ├── module.py      # the Module contract ← you extend this
│   ├── registry.py    # loads modules, gathers tools + perceptions
│   ├── robot.py       # Robot interface + MockRobot (real SDK plugs in)
│   └── brain/
│       ├── reflex.py       # the Router LLM (instant vs escalate)
│       └── agent_brain.py  # Claude Agent SDK + MCP servers
├── modules/
│   └── idle/          # example module; copy to add more (voice, vision…)
└── providers/
    └── cartesia.py    # external service template (text-to-speech)

Wake word + STT and the MCP connections slot into brain/ and a small modules/voice/; each new tool is an MCP server registered in config. The core loop doesn't change.
